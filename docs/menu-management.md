# Управление меню через админку

Документ фиксирует быстрый рабочий процесс для обновления меню, фотографий и порядка блюд.

## Главный принцип

Админка умеет:

- редактировать названия блюд;
- редактировать описания;
- менять цены по точкам `о12`, `к10`, `п7`;
- менять КБЖУ;
- менять путь к фотографии;
- менять порядок блюд кнопками `Выше` / `Ниже`;
- скачивать меню в JSON;
- загружать меню из JSON.

После любых изменений нужно нажать `Сохранить`.

## Быстрый сценарий обновления меню

1. Открыть `/admin`.
2. Нажать `Скачать JSON`.
3. Сохранить файл как резервную копию.
4. Отредактировать JSON или подготовить новый файл по шаблону `docs/menu-json-template.json`.
5. Нажать `Загрузить JSON`.
6. Проверить сводку: количество категорий и позиций.
7. Проверить меню визуально в админке.
8. Нажать `Сохранить`.
9. Открыть `/menu/o12`, `/menu/k10`, `/menu/p7` и проверить результат.

## Формат JSON

Поддерживаются два формата.

### Вариант 1: объект с `catalog`

```json
{
  "catalog": [
    {
      "category": "завтраки",
      "items": []
    }
  ]
}
```

### Вариант 2: сразу массив категорий

```json
[
  {
    "category": "завтраки",
    "items": []
  }
]
```

Рекомендуемый вариант: объект с `catalog`. Так проще добавлять служебные поля вроде `exportedAt`.

## Структура позиции

```json
{
  "id": "big-breakfast-trout",
  "name": "большой завтрак с форелью",
  "description": "форель слабой соли, яйцо, хлеб собственного производства, овощи",
  "image": "/images/menu/big-breakfast-trout.webp",
  "priceByLocation": {
    "o12": 1100,
    "k10": 1050,
    "p7": 990
  },
  "nutrition": {
    "calories": 650,
    "protein": 25,
    "fat": 30,
    "carbs": 60
  }
}
```

## Правила для `id`

`id` должен быть стабильным и написан латиницей.

Хорошо:

```txt
big-breakfast-trout
raf-basil
almond-croissant
napoleon
```

Плохо:

```txt
Большой завтрак
блюдо 1
new item!!!
```

Если меняется только название блюда, `id` лучше не менять.

## Фотографии

Для production используем путь к файлу в проекте:

```txt
/images/menu/dish-name.webp
```

Файлы должны лежать в папке:

```txt
public/images/menu/
```

Пример:

```txt
public/images/menu/big-breakfast-trout.webp
```

В JSON и админке указывается путь:

```txt
/images/menu/big-breakfast-trout.webp
```

## Требования к фотографиям

Рекомендуемый формат:

```txt
webp
1200x900 или 1600x1200
до 500-800 KB на файл
латинское имя без пробелов
```

Примеры названий:

```txt
big-breakfast-trout.webp
raf-basil.webp
almond-croissant.webp
napoleon.webp
```

Google Drive не использовать как источник картинок для сайта. Он не является стабильным CDN и часто отдаёт не изображение, а страницу предпросмотра или редирект.

## Цены по точкам

Каждая позиция должна иметь цены для всех трёх точек:

```json
"priceByLocation": {
  "o12": 1100,
  "k10": 1050,
  "p7": 990
}
```

Если позиции нет на точке, лучше временно ставить `0` и отдельно проверить отображение на сайте.

## КБЖУ

Если КБЖУ неизвестно, можно ставить нули:

```json
"nutrition": {
  "calories": 0,
  "protein": 0,
  "fat": 0,
  "carbs": 0
}
```

## Порядок блюд

Порядок блюд на сайте соответствует порядку объектов в массиве `items`.

Можно менять порядок двумя способами:

1. В JSON переставить объекты местами.
2. В админке использовать кнопки `Выше` / `Ниже`.

## Безопасный процесс перед большим обновлением

Перед большой загрузкой меню:

1. Нажать `Скачать JSON`.
2. Сохранить файл как backup.
3. Загрузить новый JSON.
4. Проверить количество категорий и позиций.
5. Проверить первые 3-5 карточек в каждой категории.
6. Нажать `Сохранить`.
7. Открыть сайт и проверить все точки.

Если что-то пошло не так, загрузить backup JSON и снова нажать `Сохранить`.
